# МонтажПаспорт

Веб-сервис на PHP 8 + SQLite для монтажников: мобильная фотофиксация работ, гарантийный PDF с QR, публичная проверка талона для клиента, отзывы и рейтинг компаний.

## Что внутри

**Для монтажника / админа компании:**
- Создание монтажа: тип работ, объект, описание работ, комментарий.
- Подсказка «что снять для этого типа работ» — сворачиваемый чек-лист рекомендуемых кадров (со звёздочкой на обязательных для гарантии).
- Загрузка фото с телефона: MIME-проверка, EXIF-поворот, JPEG-нормализация, thumbnail, теги «До / В процессе / После / Прочее».
- Поддержка HEIC (через `Imagick`, если установлен).
- Гарантийный PDF с фотоотчётом, QR-кодом и двумя кодами доступа.
- Модерация отзывов клиентов.

**Для клиента (без аккаунта):**
- `/customer.php?n=…&c=…` — открыть свой талон по QR из PDF.
- Два уровня доступа: **публичный** (по `verification_code`, ФИО/телефон маскированы под 152-ФЗ) и **личный** (по `access_token`, всё открыто + возможность оставить отзыв).
- Лендинг `/` с рейтингом компаний и поиском гарантийника по номеру.

**Для superadmin:**
- Управление компаниями (создание, отключение, бренд: ИНН, телефон, email, адрес, логотип).
- Один пользователь видит и редактирует только свою компанию; superadmin видит всё.
- Изоляция данных — пользователь компании А не имеет доступа к монтажам компании Б.

## Быстрый запуск

1. PHP 8.1+ с расширениями: `pdo_sqlite`, `gd`, `fileinfo`, `mbstring` (обязательно), `exif`, `imagick` (рекомендуется).
2. `composer install --no-dev --optimize-autoloader` (тянет `mpdf/mpdf` и `mpdf/qrcode` — последний нужен для QR в PDF, иначе генерация падает).
3. Web-root должен указывать на `public/`.
4. `storage/` доступна на запись веб-пользователю.
5. `php scripts/preflight.php` — проверка окружения.
6. Открыть `/login.php`.

При первом запуске БД создаётся автоматически, дефолтный админ `admin@example.com` со случайным паролем пишется в `storage/initial-admin-credentials.txt`. Можно задать через ENV `ADMIN_EMAIL` / `ADMIN_PASSWORD`. Этот пользователь автоматически становится **superadmin** и привязывается к компании id=1 («Моя компания»).

После апгрейда с моно-арендной версии миграция автоматически:
- создаёт компанию id=1 из существующих `app_settings` (имя, ИНН, телефон, email, адрес, логотип);
- привязывает всех пользователей и монтажи к этой компании;
- даёт первому активному админу права `is_superadmin = 1`.
