# Knowledge Cards

## C-001: Главная модель предметной области
**Тезис:** один монтаж = объект работ, внутри много элементов, у каждого элемента свои фото.
**Источник:** `architecture/domain-model.md`, ADR-0001.
**Влияние:** БД, UI, PDF-группировка, права доступа.

## C-002: Два уровня фото
**Тезис:** `common` (на объект) и `item` (на элемент) хранятся в одной таблице с полем `scope`. К каждому ещё `photo_stage` ∈ {before, during, after, other}.
**Источник:** `architecture/storage-layout.md`, `api/form-contracts.md`.
**Влияние:** чеклисты, PDF и группировка в customer-портале.

## C-003: Mobile-first как продуктовая константа
**Тезис:** UI проектируется сначала для телефона (1 колонка, крупные controls, camera capture).
**Источник:** `product/mobile-first-ux.md`.
**Влияние:** формы, загрузка фото, скорость сценария монтажника.

## C-004: Безопасность загрузки
**Тезис:** доверяем только MIME+серверной обработке, не имени файла клиента; storage вне webroot.
**Источник:** `security/file-upload-policy.md`.
**Влияние:** защита от malicious upload.

## C-005: PDF выводит только существующие фото
**Тезис:** пустые блоки запрещены; отсутствующие фото не печатаются.
**Источник:** `product/photo-checklist.md`.
**Влияние:** профессиональный вид талона.

## C-006: Multi-tenant с жёсткой изоляцией
**Тезис:** каждый `user` / `installation` принадлежит ровно одной `company`. Сессия хранит `company_id`, и все SQL-запросы фильтруют по нему. Superadmin — единственное исключение.
**Источник:** ADR-0003, `security/auth-and-sessions.md`.
**Влияние:** все запросы в `dashboard.php`, `installation_edit.php`, `users.php` обязаны фильтровать.

## C-007: Два кода доступа для customer-портала
**Тезис:** `verification_code` — публичный для QR, ПДн маскируются. `access_token` — личный, ПДн открыты + право оставить отзыв. Оба сравниваются через `hash_equals`.
**Источник:** ADR-0004, `security/threat-model.md`.
**Влияние:** PDF, customer.php, review_submit.php, photo_public.php.

## C-008: client_ip за прокси
**Тезис:** `client_ip()` возвращает `X-Forwarded-For` только если `REMOTE_ADDR` — loopback. Иначе можно подделать заголовок и обойти rate-limit.
**Источник:** `security/auth-and-sessions.md`.
**Влияние:** rate-limit на login и customer-отзывы.

## C-009: Деактивация компании — операция доступа, а не флаг
**Тезис:** `companies.is_active = 0` блокирует логин её пользователей и рвёт их активные сессии. Это не «скрытие в списке», а реальная остановка работы арендатора.
**Источник:** ADR-0003, `app/auth.php`.
**Влияние:** биллинг и саппорт смогут отключать недобросовестных арендаторов.

## C-010: TTL для audit_log
**Тезис:** `audit_log_cleanup($days)` удаляет старые записи. Триггерится лениво (1/1000) + рекомендуется cron-скрипт `scripts/cleanup.php`.
**Источник:** `app/audit.php`, `docs/operations/deploy-hestia.md`.
**Влияние:** ограничение размера БД на длинной дистанции.
