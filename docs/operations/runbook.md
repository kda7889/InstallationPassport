# Runbook

## Smoke перед запуском

### Внутренний поток
1. Открыть `/`, увидеть лендинг с топами (на чистой базе они пустые — OK).
2. Логин под дефолтным админом из `storage/initial-admin-credentials.txt`.
3. Открыть `/companies.php` (только superadmin), убедиться, что есть Default-компания id=1.
4. `/company_edit.php` — задать имя, ИНН, телефон, адрес, логотип.
5. Создать пользователя-монтажника через `/users.php`.
6. Логин под монтажником.
7. Создать монтаж: тип работ, адрес.
8. На странице монтажа развернуть блок «Что снять для этого типа работ» — там должен быть список рекомендуемых кадров (звёздочкой отмечены обязательные для гарантии).
9. Загрузить несколько фото в разных стадиях (До / В процессе / После работ).
10. Сформировать PDF, скачать, открыть в просмотрщике.
11. В PDF должны быть: фотоотчёт по стадиям, QR-код, отдельная строка с публичной ссылкой и отдельная строка с личной.

### Customer-portal
1. Сканировать QR из PDF телефоном — должен открыться `customer.php` в публичной проекции (ФИО маскированы).
2. Открыть личный URL из PDF — талон откроется в personal-режиме (ФИО видны, доступна кнопка «Оставить отзыв»).
3. Оставить отзыв, проверить, что он отображается на странице талона.
4. Войти админом, `/reviews.php` — отзыв виден, можно скрыть / показать.

### Безопасность
1. Логин под админом-А, получить чужой installation id из БД, открыть `/installation_edit.php?id=…` → 403.
2. Деактивировать компанию через `/companies.php`. Логин её пользователей блокируется, активная сессия рвётся.
3. 11+ неудачных попыток логина с одного IP → блок на 5 минут.

## Частые проблемы

### `open_basedir restriction in effect`
HestiaCP shared-сборка: `/home/<user>` — симлинк на `/srv/shared/<user>`. Дописать реальный путь в `open_basedir` пула PHP-FPM, см. `docs/operations/deploy-hestia.md` §8a.

### PDF не формируется
- `composer install --no-dev --optimize-autoloader`.
- Права записи в `storage/tmp/`.
- Логи Apache / PHP-FPM (`tail -30`).

### Не работает upload
- Проверить `upload_max_filesize`, `post_max_size`, `memory_limit`.
- `gd` или `imagick` установлен?
- Права `storage/installations/...` для веб-пользователя.

### Сломан логотип компании
- Файл должен быть в `storage/branding/company-<id>.<ext>`.
- `companies.logo_path` указывает на тот же путь относительно корня проекта.
- Получить через `/branding_logo.php?company=<id>`.

### Лендинг показывает пустоту
Нужно ≥1 отзыв на компанию, чтобы она появилась в топе. До первых отзывов отображается только поиск.

### `audit_log` разрастается
- Ленивая чистка: `random_int(1,1000)==1` на каждый insert триггерит `audit_log_cleanup(365)`.
- Принудительно: `php scripts/cleanup.php 365`.
- В cron: см. §10 в `deploy-hestia.md`.

## Резервное восстановление

1. Остановить PHP-FPM пул.
2. Залить бэкап `storage/` поверх (БД + фото + PDF).
3. Запустить пул. Миграции применятся автоматически при первом запросе через `db_bootstrap_if_needed()`.
